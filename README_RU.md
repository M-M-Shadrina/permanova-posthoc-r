## Описание

Интерактивный R-скрипт для проведения **PERMANOVA** (`adonis2`, пакет vegan)  
с последующим **попарным пермутационным t-тестом** (post-hoc анализ)  
одновременно по нескольким зависимым переменным.  
Результаты экспортируются в файл **Excel** с четырьмя листами.

## Требования

| Пакет | Назначение |
|-------|------------|
| `tidyverse` | Обработка данных |
| `vegan` | PERMANOVA (`adonis2`) |
| `RVAideMemoire` | Попарный пермутационный t-тест |

```r

                   "effectsize", "openxlsx"))
```

Протестировано на R 4.3+

## Формат входных данных

CSV-файл, содержащий:

- **Столбцы-факторы** — категориальные переменные группировки (например, `Group`, `Gender`)
- **Столбцы-переменные** — числовые зависимые переменные (например, объёмы структур мозга)

Поддерживаемые разделители: `,` (запятая) или `;` (точка с запятой).

Пример структуры (`table_example.csv`):

![Пример входной таблицы](https://github.com/M-M-Shadrina/permanova-posthoc-r/blob/main/Example%20input%20table.png)

Таблица содержит столбцы-факторы (`Group`, `Age_Group`, `Gender`) и множество числовых переменных (например, объёмы структур мозга).

## Запуск

1. Откройте R или RStudio
2. Запустите скрипт:

```r
source("permanova_pipeline_ru.R")
```

3. Следуйте интерактивным подсказкам:

| Шаг | Подсказка |
|-----|-----------|
| 1 | Путь к CSV-файлу |
| 2 | Разделитель (`1` = запятая, `2` = точка с запятой) |
| 3 | Количество факторов (2 или 3) и их названия |
| 4 | Количество зависимых переменных и их названия |
| 5 | Подтверждение списка переменных |
| 6 | Путь для сохранения (Enter = текущая папка) |
| 7 | Метод поправки на множественные сравнения |

---

## Статистическая логика

```
Для каждой зависимой переменной:
│
├── PERMANOVA (adonis2, евклидово расстояние, 9 999 пермутаций)
│     Модель: переменная ~ фактор1 * фактор2 [* фактор3]
│     Результат: Df, SumOfSqs, R², F, p-значение, звёздочки
│     Поправка FDR по всем переменным (метод BH)
│
└── Post-hoc: попарный пермутационный t-тест (RVAideMemoire)
      Проводится внутри каждого уровня фактора 2
      Размер эффекта: d Коэна (объединённое SD)
      Поправка на множественные сравнения: на выбор пользователя
      Поправка FDR по всем сравнениям (метод BH)
```

## Результаты

Скрипт сохраняет файл `PERMANOVA_results.xlsx` с четырьмя листами:

| Лист | Содержимое |
|------|------------|
| `Primary_all` | Полная таблица PERMANOVA по всем переменным |
| `Primary_significant` | Строки с FDR < 0.05 |
| `PostHoc_all` | Все попарные сравнения |
| `PostHoc_significant` | Попарные сравнения с FDR < 0.05 |

Обозначения: `*` p < 0.05 · `**` p < 0.01 · `***` p < 0.001

### Описание столбцов — основные листы (Primary)

| Столбец | Описание |
|---------|----------|
| `Structure` | Название переменной (измерения) |
| `Term` | Член модели (фактор, взаимодействие, Residual, Total) |
| `Df` | Степени свободы |
| `SumOfSqs` | Сумма квадратов |
| `R2` | Доля объяснённой дисперсии |
| `F` | F-статистика |
| `p_value` | Сырое p-значение (на основе перестановок) |
| `stars` | Звёздочки по сырому p |
| `FDR` | p-значение с поправкой BH по всем переменным |
| `stars_FDR` | Звёздочки по FDR |

![Результаты PERMANOVA](https://github.com/M-M-Shadrina/permanova-posthoc-r/blob/main/output_primary_all.png)

*Лист `Primary_all`: результаты PERMANOVA для каждой переменной и терма — R², F-статистика, p-значение и значимость с поправкой FDR.*

### Описание столбцов — листы post-hoc

| Столбец | Описание |
|---------|----------|
| `Structure` | Название переменной |
| `Comparison` | Подпись попарного сравнения |
| `Cohen_d` | Размер эффекта (d Коэна) |
| `p_value` | Сырое p-значение |
| `stars` | Звёздочки по сырому p |
| `FDR` | p-значение с поправкой BH |
| `stars_FDR` | Звёздочки по FDR |

![Результаты post-hoc](https://github.com/M-M-Shadrina/permanova-posthoc-r/blob/main/output_PostHoc.png)

*Лист `PostHoc_all`: попарные сравнения внутри каждого уровня фактора 2 — размер эффекта d Коэна, p-значение и поправка FDR.*

## Примечания

- Переменные с менее чем 12 наблюдениями автоматически пропускаются.
- Если для какой-либо переменной возникает ошибка в `adonis2` или post-hoc тесте, переменная пропускается, анализ продолжается.
- **Пользователи Windows:** можно вставлять путь с кавычками — они удаляются автоматически.
